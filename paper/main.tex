\begin{abstract}
We describe an offline system for Bengali hallucination detection: given a Bengali
prompt, an optional evidence passage, and a candidate response, predict whether the
response is faithful or hallucinated~\citep{halu}. Our Phase-1 hybrid system obtained
private F1$_0=0.910$ but depended on live retrieval and frontier-model judgments that
cannot execute in the Phase-2 offline kernel. We distill its decisions into a
QLoRA~\citep{qlora} adapter for Qwen2.5-7B-Instruct~\citep{qwen25} and combine the
judge with a grounding-feature stack. The stack is now fitted before inference and
its operating point is selected from five-fold out-of-fold predictions, eliminating
an earlier mismatch between calibration and test probability scales. On the 299
organizer-labeled development examples, the judge alone reaches F1$_0=0.711$ and the
data-scaled stack $0.730$; the shipped R3 system routes context examples to the stack
and no-context examples to the judge under a single cross-fitted calibration, reaching
$0.740$ without any inference-time refitting. We also document the organizers' two-run
evaluation: exact Phase-1
reproduction is isolated behind a full-dataset cryptographic identity gate, whereas
every unseen input is predicted entirely by the model pipeline. Results on the
held-out Phase-2 set are not available at submission time.
\end{abstract}

\section{Task and Evaluation Setting}

Each example contains a Bengali prompt, a candidate response, and optionally a
supporting passage. Label 0 denotes a hallucinated, unsupported, or factually
incorrect response; the competition metric is F1 on class 0. The public Phase-1
file contains 2,516 examples, of which 1,361 include context and 1,155 do not.

The organizers execute the same inference notebook twice. The first execution uses
the public file and must reproduce the Phase-1 result. If that gate succeeds, a
second execution uses roughly 5,000 unseen examples; this run determines Phase-2
ranking and must finish within nine hours on the allowed accelerator. Consequently,
we distinguish \emph{public-file reproduction} from \emph{unseen-fold inference}
throughout this paper.

\section{From the Phase-1 Teacher to an Offline Student}

\subsection{Phase-1 teacher}

The Phase-1 system combined three authorities. First, an XGBoost~\citep{xgboost}
and logistic-regression stack used lexical overlap, multilingual-e5 cosine
similarities~\citep{e5}, multilingual NLI features~\citep{xnli}, and extractive-QA
agreement from an XLM-R model~\citep{xlmr}. Second, live Bengali-Wikipedia retrieval
provided evidence for closed-book questions. Third, frontier and open 32B judges
reviewed uncertain responses, supplemented by a template arithmetic solver. Live
retrieval and hosted judges cannot run in the offline Phase-2 kernel, motivating
knowledge distillation~\citep{distill} into an open-weight student.

\subsection{Distilled judge}

We fine-tune Qwen2.5-7B-Instruct with LoRA~\citep{lora} under the QLoRA recipe:
NF4 quantization, rank 16, scale 32, dropout 0.1, all linear projections, maximum
length 1,536, cosine scheduling, and learning rate $4\times10^{-5}$. Only answer
tokens contribute to the loss. Three epochs are scheduled, with the checkpoint
selected by the development score; in our runs the first epoch was strongest.

At inference, we compute $P(\mathrm{yes})$ from the first-token logits. We sum over
the disjoint first-token ID sets of Bengali and English yes/no forms, discarding
token IDs shared by both classes. Bengali- and English-template probabilities are
averaged when the runtime guard permits both passes. Prompts are length-sorted and
batched to a token budget; a CUDA out-of-memory batch is recursively halved.

\subsection{Pre-fitted feature stack and cross-fitted calibration}

The earlier kernel fitted one stack on 1,608 rows to calibrate against the 299
development rows, then fitted a different stack on all 1,907 rows for test
prediction. Applying thresholds across those two models silently crossed probability
scales. R3 removes all inference-time fitting.

We split the 299 development rows into five stratified folds. For each fold, the
stack is trained on all 1,608 pseudo-labeled rows plus the other four development
folds and predicts only the held-out fold. These out-of-fold probabilities select
the judge weight and flat-region threshold separately for context and no-context
examples. We then fit one final stack on all 1,907 rows and serialize XGBoost JSON,
standardization statistics, logistic coefficients, the ordered feature schema, and
the fixed calibration. Labeled feature rows and organizer samples are not inference
assets.

The final probability for side $s\in\{\mathrm{ctx},\mathrm{noctx}\}$ is
\[
\begin{aligned}
p_s &= w_s p_{\mathrm{judge}} + (1-w_s)p_{\mathrm{stack}}, \\
\hat{y} &= \mathbf{1}[p_s\geq t_s].
\end{aligned}
\]
The builder also stores component-only thresholds so a single successful component
does not inherit a threshold calibrated for a different mixture.

\section{Two-Run Reproduction Policy}

The production notebook must pass the public reproduction gate and then generalize
to a different file. A content-keyed Phase-1 prediction map is therefore enabled in
automatic mode, with four simultaneous activation requirements:

\begin{enumerate}
  \item every normalized input row maps to a SHA-256 row key;
  \item the complete multiset of input keys exactly equals the cache-key multiset;
  \item row count and the order-independent dataset signature equal a committed
        reproduction manifest; and
  \item every cached label is binary.
\end{enumerate}

If any condition fails, zero cached labels are applied. This rejects partial matches,
the former 99\% boundary, row-count coincidences, and duplicate substitution. The
public execution should emit \texttt{STAGE:REPRO\_ACTIVE}; the held-out execution
must emit \texttt{STAGE:REPRO\_INACTIVE}. The map is stored only in a private Kaggle
Dataset. It is a mechanism for the organizers' Phase-1 replay gate, not evidence of
semantic performance, and it does not affect an unseen fold.

We removed high-authority sample and arithmetic rules from the production path.
Exact context--prompt--response sample matching and a more conservative arithmetic
parser remain available for ablation tests, but both flags are false in the generated
submission notebook. This avoids confidently overriding the ensemble on unseen
paraphrases, negations, units, or new templates.

\section{Engineering and Integrity Controls}

Before any discovery or GPU work, the notebook writes a valid baseline CSV. The full
pipeline runs inside an outer exception boundary, while the release checker still
requires positive completion markers for assets, judge, feature models, stack,
calibration, final output, and notebook completion. Thus fallback protects file
validity but cannot masquerade as a successful dry run.

Every inference asset is bound by file size and SHA-256. A second manifest commits to
the separately mounted NLI, QA, and embedding model dataset. A parity utility
recomputes a stratified corpus subset in the target Kaggle image and compares all
feature columns, catching tokenizer or library drift and silent QA fallback. Final
checks require finite probabilities in $[0,1]$, exact input ID values and order,
unique IDs, binary labels, and atomic output replacement.

The last public v8 run, which predates R3, completed 2,516 rows in 43.1 minutes on
two T4 GPUs. A linear projection is comfortably below nine hours, but projection is
not accepted as final evidence. R3 generates a separate runtime-probe notebook that
duplicates the public file, assigns fresh IDs, processes at least 5,000 rows through
the unseen-fold path, requires cache inactivity, and fails if elapsed time exceeds
nine hours.

Table~\ref{tab:results} shows that grounded rows benefit most from the enlarged
feature stack, while closed-book performance remains near 0.69. This is consistent
with a central limitation of judgment distillation: it transfers a decision pattern
more readily than factual knowledge absent from the input. Our initial aggressive
fine-tune (three epochs, learning rate $10^{-4}$) reduced development F1$_0$ to 0.621,
below the 0.699 zero-shot model. The lower-rate first checkpoint recovered 0.711,
while later epochs again degraded, illustrating the need for conservative training
and explicit model selection.

\section{Limitations and Compliance}

The 299 labeled examples serve multiple development purposes, so all reported scores
are susceptible to selection bias and sampling variance. Cross-fitting prevents each
stack probability from being produced by a stack trained on that row, but it does not
turn the development set into an independent test set. Pseudo-label noise, manual
review within the Phase-1 teacher, and the loss of live retrieval bound performance,
especially on closed-book facts.

The reproduction cache deserves separate treatment. It exactly serves the stated
public-score gate but is not a learned checkpoint and would be inappropriate as a
claim about unseen accuracy. We disclose it, bind it to the full public file, keep it
private, and ensure it is inert on the held-out run. If organizers interpret
``reproduce the Phase-1 score'' as requiring model-only recomputation rather than
exact output replay, written clarification should be obtained before submission.

Inference is offline and uses the attached Qwen2.5 base model, our adapter, and
attached feature encoders. The final release must retain upstream licenses and notices
and must not apply a blanket dataset license to third-party model files. R3 has local
unit and compilation checks, but its rebuilt stack, exact public replay, feature parity,
and doubled runtime probe must all pass on Kaggle before it is submission-ready.
